Generics
=========
Arrays are typesafe.
We can provide guarantee that what type of elements are present in array.
ex:
	int[] arr = new int[3];
	arr[0]=10;
	arr[1]=20;
	arr[2]=30;

If requirement is there store string values then we need to use String[] array. 
ex:
	String[] s = new String[10];
	s[0]="Hi";
	s[1]="Bye";
	s[2]=10; // C.T.E 

At the time of retrieving the data we don't need to perform typecasting.
ex:
	String[] s = new String[10];
	s[0]="Hi";
	s[1]="Bye";
	-
	-
	String val1 = s[0];

Collections are not typesafe.

We can't provide guarantee that what type of elements are present in collections.

If requirement is there to store string values then it is not recommanded to use ArrayList class because we won't get any compile time error or runtime error but sometimes our program get failure.
ex:
	ArrayList al =new ArrayList();
	al.add("Hi");
	al.add("Bye");
	al.add(10);

At the time of retrieving the data compulsary we need to perform typecasting.
ex:
	ArrayList al =new ArrayList();
	al.add("Hi");
	al.add("Bye");
	al.add(10);
	-
	-
	String s =(String)al.get(0);

To overcome this limitation Sun Micro System introduced Generics concept in 1.5 version.

The main objective of Generics is 

1) To make Collections as typesafe.

2) To avoid typecasting problem.

Collection

If we want to represent group of individual objects in a single entity then we need to use Collection.

It is a child interface of Collection interface.

If we want to represent group of individual objects in a single entity where duplicate objects are allowed and order is preserved then we need to use List.

List.of()
=========
It is introduced in Java 9.
It gives immutable object.
We can't add, remove and replace the elements.
It does not allow null.

ex:
--
import java.util.*;
class Test  
{
	public static void main(String[] args) 
	{
		List<Integer> list = List.of(4,8,1,3,9,6);
		
		for(int i : list)
		{
			System.out.print(i+" ");
		}
	}
}

Interview Question 
==================
Q) Write a java program to display reverse of a string?

input:
	dog is animal

output:
	animal is dog 

Approach1
---------
import java.util.*;
class Test  
{
	public static void main(String[] args) 
	{
		String str= "dog is animal";
		String[] sarr = str.split(" ");
		for(int i=sarr.length-1;i>=0;i--)
		{
			System.out.print(sarr[i]+" ");
		}
	}
}

Approach2
--------
import java.util.*;
class Test  
{
	public static void main(String[] args) 
	{
		String str= "dog is animal";
		List<String> list = Arrays.asList(str.split(" "));
		Collections.reverse(list);
		for(String s : list)
		{
			System.out.print(s+" ");
		}
	}
}

ArrayList 
========
The underlying data structure is resizable array or growable array.

Duplicate objects are allowed.

Insertion order is preserved.

Hetrogeneous objects are allowed.

Null insertion is possible.

It implements List, Serializable, Cloneable and RandomAccess interface.

ex:
---
import java.util.*;
class Test  
{
	public static void main(String[] args) 
	{
		ArrayList al =new ArrayList();
		al.add(7);
		al.add(2);
		al.add(9);
		System.out.println(al);//[7,2,9]
		
		al.add(7);
		System.out.println(al);//[7,2,9,7]
		
		al.add("Hi");
		System.out.println(al);//[7,2,9,7,Hi]
		
		al.add(null);
		System.out.println(al);//[7,2,9,7,Hi,null]
	}
}
