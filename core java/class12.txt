Q) What is the difference between System.out.println() and System.err.println()?

System.out.println()
--------------------
It is used to display standard output.
It is used to display the output only on console.

System.err.println()
--------------------
It is used to display error output.
It is used to display the output on console as well as it redirects to physical file.
ex:
	class Test 
	{
		void main()
		{
			System.err.println("Error Output");		
		}
	}
	javac  Test.java
	java   Test 2>demo.txt

Fully Qualified Name 
====================
Fully qualified name means we will declare a class or interface along with package name.

It improves readability of our code.
ex:
	java.io.File (C)
	java.util.Enumeration (I) 

ex:
---
class Test 
{
	void main()
	{
		java.util.Date d = new java.util.Date();
		System.out.println(d); 		
	}
}

In realtime we will use fully qualified name when we are using a class or interface which is present in two different packages and we required both the classes or interface in our program.
ex:
	java.util.Date 
	java.sql.Date 	


Import statements 
=================
Whenever we use import statement we should not use fully qualified name.

Using short name also we can achieve.

In java, we have three import statements.

1) Explicit class import 

2) Implicit class import 

3) Static import 

1) Explicit class import
-------------------------
This type of import statement is highly recommanded to use because it increase readability of our code.

ex:
---
import java.time.LocalDate;
import java.time.LocalTime;
class Test 
{
	void main()
	{
		LocalDate d = LocalDate.now();
		System.out.println(d); 

		LocalTime t = LocalTime.now();
		System.out.println(t); 
	}
}

2) Implicit class import 
-------------------------
This type of import statement is not recommanded to use because it will reduce readability of our code.

ex:
---
import java.time.*;
class Test 
{
	void main()
	{
		LocalDate d = LocalDate.now();
		System.out.println(d); 

		LocalTime t = LocalTime.now();
		System.out.println(t); 
	}
}
